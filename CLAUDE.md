# CLAUDE.md — правила для Claude Code в Asya Local

Этот файл задаёт краткие рабочие правила для Claude Code в репозитории `ajotce/asya-assistant`.

## Обязательный старт перед задачей
1. Прочитать `README.md`.
2. Прочитать `AGENTS.md`.
3. Прочитать актуальные документы в `docs/` по теме задачи.

## Принципы работы
1. Работать маленькими проверяемыми шагами.
2. Сначала изучить текущую реализацию, потом менять код.
3. Не менять архитектуру без явной причины.
4. Не добавлять лишние зависимости без необходимости.

## Документация и проверки
1. При изменении поведения обновлять документацию.
2. Запускать доступные проверки и тесты.
3. Если проверки не запущены, явно указать причину.

## Безопасность
1. Не коммитить `.env`, токены и другие секреты.
2. Не выводить API-ключи в коде, логах и пользовательских сообщениях.
3. Использовать только `.env.example` для примеров переменных.

## Коммуникация
1. Писать пользователю по-русски, простым и понятным языком.
2. В конце задачи кратко сообщать, что сделано, какие файлы изменены и какие проверки выполнены.
