# CLAUDE.md

> Этот файл — тонкая обёртка для Claude Code. Все основные правила работы — в `AGENTS.md`. Здесь только Claude-специфика.

## Главное

Прочитай `AGENTS.md` перед началом работы. Он — единственный источник правды о проекте, текущей фазе, запретах и стиле работы.

После `AGENTS.md` прочитай `docs/roadmap.md` и файлы `docs/acceptance/v0.X.md` для текущей фазы.

## Claude-специфика

### TodoWrite для отслеживания задач

Когда задача состоит из нескольких шагов (3+), используй TodoWrite, чтобы видеть прогресс. Помечай шаги как `in_progress` перед началом и `completed` сразу после завершения, не накапливай статусы.

### Plan mode для нетривиальных изменений

Перед любым изменением, которое затрагивает больше двух файлов или меняет архитектуру, переходи в plan mode и согласовывай план с пользователем до начала кода.

### Extended thinking при сложных решениях

Когда задача связана с архитектурным выбором (например, как организовать миграцию данных, какой паттерн применить) — думай развёрнуто перед ответом, не торопись.

### Чтение перед редактированием

Перед редактированием файла обязательно читай его целиком (или релевантные части), даже если изменение кажется тривиальным. Маленькие правки часто ломают невидимый контекст.

### Не запускай разрушающие команды без подтверждения

`rm -rf`, `git push --force`, `DROP TABLE`, `alembic downgrade base` на боевой БД — всегда сначала спрашивай. Даже если задача формально это подразумевает.
