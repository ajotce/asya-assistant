# Onboarding (1.0.11)

## Цель

Onboarding показывается при первом входе пользователя (`user.onboarding_completed=false`) и помогает быстро
запустить базовый сценарий использования Asya.

## Экраны

1. Приветствие и назначение продукта.
2. Кратко: ключевые возможности Asya.
3. Рекомендация подключить первую интеграцию.
4. Подсказка по выбору имени/голоса ассистента в Settings.
5. Финальный экран и переход в Base-chat.

На каждом шаге пользователь может двигаться вперед/назад.

## Пропуск

Кнопка `Пропустить` доступна на каждом шаге.
И завершение, и пропуск вызывают `POST /api/auth/onboarding/complete`, после чего:

- `user.onboarding_completed` становится `true`;
- пользователь переходит в основной интерфейс (Base-chat).

## Технические детали

- Backend поле: `users.onboarding_completed` (`boolean`, default `false`).
- Backend endpoint: `POST /api/auth/onboarding/complete`.
- Frontend условие показа: если `auth/me` вернул `onboarding_completed=false`.

## Как протестировать вручную

1. Создать нового пользователя (signup/OAuth callback в разрешенном режиме).
2. Войти, убедиться что показывается onboarding.
3. Пройти до конца: проверить переход в чат.
4. Повторно открыть приложение: onboarding больше не показывается.
5. Для отдельной проверки skip: создать нового пользователя и нажать `Пропустить`.
