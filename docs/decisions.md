# Architecture Decisions (MVP)

Дата: 2026-04-25

## ADR-001: Без долгосрочной памяти
Решение: в MVP хранить только временный контекст сессии backend.
Причина: соответствует ограничениям MVP и снижает риски хранения персональных данных.

## ADR-002: API-ключ только на backend
Решение: `VSELLM_API_KEY` хранится только в `.env` backend и никогда не отдается во frontend.
Причина: безопасность и требования ТЗ.

## ADR-003: Без web-search в MVP
Решение: не добавлять веб-поиск.
Причина: это явно вне объёма MVP.

## ADR-004: Локальный режим через backend static serving
Решение: основной режим MVP — frontend раздается backend-ом локально.
Причина: избежать проблем HTTPS GitHub Pages -> локальный HTTP backend.

## ADR-005: Временная локальная векторная база
Решение: embeddings и индекс файлов хранятся только в рамках активной сессии.
Причина: соблюдение запрета на долговременную память в MVP.

## ADR-006: Без авторизации в MVP
Решение: авторизацию не добавлять в MVP, оставить архитектурную возможность на будущее.
Причина: локальный режим и требование минимального объема.

## ADR-007: Единый префикс backend API
Решение: использовать `/api/*` как основной путь endpoint-ов.
Причина: единообразие API и соответствие техническому заданию и плану.
