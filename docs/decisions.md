# Decisions (MVP)

Дата обновления: 2026-04-25

## ADR-001: Работа строго в рамках MVP
Решение: не добавлять функции вне текущего этапа плана разработки.
Причина: снизить риск усложнения и ускорить выпуск стабильного MVP.

## ADR-002: API-ключ только на backend
Решение: `VSELLM_API_KEY` хранится только в `.env` на стороне backend.
Причина: безопасность и соответствие ТЗ.

## ADR-003: Временный контекст, без долгосрочной памяти
Решение: хранить контекст только в рамках текущей backend-сессии.
Причина: это прямое ограничение MVP.

## ADR-004: Локальный запуск — основной режим MVP
Решение: backend запускается локально в Docker, frontend работает в связке с ним.
Причина: это целевой режим текущего этапа.
