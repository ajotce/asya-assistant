# Decisions (Asya Local)

Дата обновления: 2026-04-27

## ADR-001: Локальный режим — основной рабочий режим
Решение: backend запускается локально в Docker, frontend работает в связке с ним и при необходимости раздаётся backend-сервисом.
Причина: это текущий стабильный режим использования проекта.

## ADR-002: API-ключ только на backend
Решение: `VSELLM_API_KEY` хранится только в `.env` на стороне backend.
Причина: ключ не должен попадать во frontend, логи и клиентский трафик.

## ADR-003: Сессионный runtime-контекст
Решение: контекст чата, файлы, embeddings и usage хранятся в runtime-сессии и очищаются при удалении сессии.
Причина: предсказуемое поведение и простой контроль жизненного цикла данных.

## ADR-004: Разделение frontend и backend
Решение: UI-слой и backend API развиваются отдельно, бизнес-логика не размещается в UI-компонентах.
Причина: упрощение поддержки, тестирования и изменений.
