# Personality (Asya 0.3)

Документ описывает базовую управляемую личность Asya в рамках версии 0.3.

## Цель
Добавить устойчивый стиль ассистента, который учитывает предпочтения пользователя и контекст пространства, без имитации «живого сознания».

## Компоненты
- `AssistantPersonalityProfile` (base): базовый профиль личности ассистента для пользователя.
- `PersonalityOverlay` (per-space): локальные корректировки стиля в пространстве.
- `BehaviorRule`: прикладные правила ответа (тон, формат, запреты, предпочтения).

## Параметры профиля (реализовано)
- имя ассистента;
- тон;
- `humor_level` (0..2);
- `initiative_level` (0..2) в рамках чата;
- `can_gently_disagree` (мягко возражать при рисках);
- `address_user_by_name` (обращаться по имени пользователя);
- `style_notes`.

## Что входит в 0.3
- адаптация тона под пользователя;
- вежливая и последовательная манера;
- лёгкий юмор, если уместно;
- мягкое возражение при риске ошибки;
- извинение при ошибке и предложение сохранить правило.

## Что не входит в 0.3
- сложная автономная эволюция личности;
- моделирование эмоций, сознания, «сущности»;
- терапевтические паттерны поведения без явного запроса.

## Требования безопасности
- личностные настройки должны быть user-scoped;
- space overlay должен применяться только внутри текущего пространства;
- никакие скрытые данные пользователя не должны попадать в публичные ответы.

## Использование в prompt (реализовано)
- Base personality добавляется в memory-context для `POST /api/chat/stream`, если профиль активен.
- Space overlay добавляется только если чат привязан к пространству и `personality_overlay_enabled=true`.
- В prompt попадают только компактные поля (`name`, `tone`, levels/flags, `style_notes` с обрезкой по длине), без внутренних ID.
- Если пользователь явно указывает, что Asya ошиблась/ответила не так, в контекст добавляется инструкция предложить сохранить правило на будущее.

## Frontend UI личности (реализовано)

Во вкладке `Память` добавлен блок `Личность Asya` с редактированием:
- имя ассистента;
- тон;
- уровень юмора;
- краткость/подробность (через `initiative_level`);
- мягкое возражение;
- обращение по имени;
- заметки по стилю.

В карточке показываются `created_at/updated_at`, а при ошибке сохранения выводится user-friendly сообщение без технических ID.

Профиль сохраняется через `PUT /api/personality`.
