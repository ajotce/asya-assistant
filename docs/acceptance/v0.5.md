## Блокеры на 2026-05-05

- Отдельной ветки `0.5/file-storage-providers` нет, реализация вошла в интеграционную ветку без выделенного branch-name.
- Часть ожидаемых feature-веток не содержит уникальных коммитов относительно текущей `0.5-extended`,
  поэтому для оценки используется фактическая реализация в коде, а не только наличие одноимённой ветки.

## Матрица branchless-реализаций (проверено по коду)

- `file-storage-providers`: реализовано в общем integration-layer (provider enum + `/api/integrations` + storage intents).
- `bitrix24-readonly`: реализовано через `/api/integrations/bitrix24/*` и read-only intents в action router.
- `imap-mail`: реализовано через `/api/integrations/imap/*` и UI в Settings.
- `observer-state-snapshots`: сведено merge-коммитом в `0.5-extended`.
- `action-rollback`: реализовано для memory snapshots (`/api/memory/snapshots/{id}/rollback`) и router intent `/rollback`.

Примечание: `document-templates`, `briefings`, `memory-personality-evolution` имеют частичную реализацию
в коде (интенты/модели/существующие 0.3-механизмы), но как отдельные закрытые v0.5-scope пока не подтверждены.

## Финальное решение по PR v0.5 -> main

- Входит в PR:
  - GitHub read-only;
  - Bitrix24 read-only;
  - IMAP-readonly;
  - file-storage providers foundation;
  - migration chain/one-head;
  - security fixes по логированию секретов;
  - QA rerun с green quality gates (py312 backend + frontend lint/test/build).
- Не входит (deferred):
  - полноценный product scope `document-templates`;
  - полный product scope `briefings`;
  - полная консолидация всех planned v0.5 инициатив как отдельных завершённых feature-треков.

Release recommendation для текущего PR-транша: `ready`.
