# Acceptance v0.5

Документ фиксирует критерии приёмки фазы `v0.5 — Extended integrations and documents`.

## 1) Процесс и ветвление

- [ ] Базовая ветка фазы: `0.5-extended`.
- [ ] Разработка ведётся через feature-ветки от `0.5-extended`.
- [ ] Нет прямых merge в `main`, пока не завершена фаза v0.5.
- [ ] Для каждой feature-ветки есть понятный scope и проверяемый результат.

## 2) Интеграции v0.5 (расширение)

- [ ] Реализован IMAP-readonly flow (подключение, чтение, безопасное хранение секретов).
- [ ] Реализован GitHub readonly flow в рамках agreed scope.
- [ ] Реализован Bitrix24 readonly flow в рамках agreed scope.
- [ ] Реализованы подключения к `Yandex.Disk`, `iCloud Drive`, `OneDrive` в рамках agreed scope.
- [ ] Для каждого нового provider соблюдён user isolation и единая OAuth/credential abstraction.

## 3) Документы и шаблоны

- [ ] Есть каталог/модель шаблонов документов в продукте.
- [ ] Есть генерация документов по шаблону с пользовательскими данными.
- [ ] Поддержан экспорт минимум в DOCX и PDF для agreed сценариев.
- [ ] Файлы и артефакты не нарушают политику безопасности (секреты/PII/logging).

## 4) Observer / activity / workflows

- [ ] Расширения observer из v0.5 roadmap внедрены без регрессий текущих v0.4 сценариев.
- [ ] Новые автоматические действия имеют явную модель подтверждений/отката там, где это требуется.
- [ ] Activity log отражает новые пользовательские действия без утечки секретов.

## 5) Документация

- [ ] Актуальны `AGENTS.md`, `CLAUDE.md`, `OPENCODE.md` для v0.5.
- [ ] Обновлены `docs/api.md`, `docs/architecture.md`, `docs/integrations.md`, `docs/security.md` по факту изменений.
- [ ] В `docs/decisions.md` зафиксированы ключевые ADR фазы v0.5.
- [ ] В `docs/development-log.md` есть записи по завершённым крупным задачам.

## 6) Качество

- [ ] Backend: `pytest backend/tests`, `ruff check`, `mypy backend`.
- [ ] Frontend: `npm run lint`, `npm test`, `npm run build`.
- [ ] Migration checks пройдены при изменениях схемы БД.
- [ ] Docker smoke (`docker compose up`, `/api/health`, `docker compose down`) пройден перед финальным merge.

## 7) Релизная готовность фазы

- [ ] Все пункты `docs/acceptance/v0.5.md` отмечены как выполненные.
- [ ] Нет открытых критичных blocker-задач по v0.5 scope.
- [ ] Документация и код синхронизированы, нет противоречий по текущей фазе.
