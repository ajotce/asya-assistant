# Asya - Memory and Personality (v0.3) Acceptance Criteria

## Overview
This document contains the acceptance criteria for version v0.3 of the Asya project, codenamed "Memory and personality".

## Criteria List

### 1. Database Foundation
- [x] SQLAlchemy models are created for: `Space`, `SpaceMemorySettings`, `UserProfileFact`, `MemoryEpisode`, `MemoryChunk`, `BehaviorRule`, `AssistantPersonalityProfile`, `MemoryChange`, `MemorySnapshot`, `ActivityLog`.
- [x] Alembic migration `20260502_04_memory_spaces_foundation.py` exists and applies successfully.
- [x] All models include `user_id` foreign key for data isolation.

### 2. Backend API
- [x] `GET /api/spaces`, `POST /api/spaces`, `GET /api/spaces/{id}`, `PATCH /api/spaces/{id}`, `DELETE /api/spaces/{id}` endpoints are working.
- [x] `GET /api/spaces/{id}/settings`, `PUT /api/spaces/{id}/settings` endpoints are working.
- [x] Memory API endpoints are working: facts (`GET`, `POST`, `PATCH`), rules (`GET`, `POST`, `PATCH`), episodes (`GET`), changes (`GET`).
- [x] Snapshot API endpoints are working: `POST /api/memory/snapshots`, `GET /api/memory/snapshots`, `GET /api/memory/snapshots/{id}`, `POST /api/memory/snapshots/{id}/rollback`.
- [x] `GET /api/activity-log` endpoint is working.

### 3. Core Logic and Services
- [x] `SpaceService` automatically creates `Base` and `Admin` (if applicable) spaces on user creation.
- [x] `MemoryService` handles operations and properly generates `MemoryChange` and `ActivityLog` records.
- [x] `MemoryExtractionService` safely extracts facts and rules from dialogs if `MEMORY_EXTRACTION_ENABLED=true`.
- [x] Snapshot creation captures all relevant memory structures (facts, rules, personality, settings).
- [x] Snapshot rollback properly restores previous memory state and generates a rollback activity event.

### 4. Chat Integration
- [x] `ChatService.stream_chat` properly includes memory context before file retrieval context.
- [x] Memory retrieval logic checks `space_memory_settings.memory_read_enabled` flag.
- [x] After sending a response, the system generates an activity event `memory_used_in_response` if memory was accessed.
- [x] Background memory extraction runs automatically after assistant responses.

### 5. Frontend UI
- [x] Memory Page shows facts, rules, and allows manual additions/edits.
- [x] Snapshot management UI allows creating and restoring snapshots.
- [x] Space management is available with per-space memory settings toggles.
- [x] Activity Log page shows chronological history of memory events.

### 6. Testing
- [x] Unit tests for memory extraction pass successfully.
- [x] Unit tests for memory API operations pass successfully.
- [x] Unit tests verify user isolation (cross-user memory access returns 404).
- [x] Frontend tests for `MemoryPage` and `ActivityPage` pass successfully.

### 7. Documentation
- [x] `docs/memory.md` is updated.
- [x] `docs/api.md` is updated with memory endpoints.
- [x] `docs/architecture.md` is updated to reflect new DB schema.
- [x] `docs/spaces.md` and `docs/activity-log.md` reflect current features.
