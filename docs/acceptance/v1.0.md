# Acceptance v1.0

Документ фиксирует критерии приёмки фазы `v1.0 — Voice & Public Release`.

Статус: в подготовке, рабочая ветка `1.0-public`.

## 1) Cloud-readiness

- [ ] В коде нет локальных путей, всё через env-vars и конфиг.
- [ ] `/healthz` отвечает `200` в runtime-окружении.
- [ ] Логи пишутся в stdout/stderr, файловые логи не обязательны.
- [ ] Для новых фич отсутствуют hardcoded machine-specific конфиги.

## 2) PostgreSQL миграция

- [ ] Alembic-миграции проходят на PostgreSQL без ошибок.
- [ ] `sqlite-vec` заменён на `pgvector`.
- [ ] Скрипт миграции данных SQLite → PostgreSQL работает на тестовой БД.
- [ ] Проверена целостность данных после миграции на тестовом наборе.

## 3) S3-хранилище

- [ ] Round-trip upload/download работает через MinIO в dev.
- [ ] Новые фичи не используют локальный диск как основное хранилище.
- [ ] Конфигурация S3 полностью через env-vars.

## 4) Облачное развёртывание

- [ ] Asya доступна по новому домену в целевом окружении.
- [ ] CI/CD выполняет автоматический deploy целевой ветки.
- [ ] Terraform описывает всю целевую инфраструктуру.

## 5) Мониторинг

- [ ] Sentry фиксирует runtime-ошибки backend/frontend.
- [ ] Grafana-dashboard показывает базовые метрики (RPS, latency, error rate, DB).
- [ ] Критичные алерты приходят в согласованный канал.

## 6) Резервное копирование

- [ ] Регулярные бэкапы создаются автоматически.
- [ ] Ручное восстановление на staging проходит полностью.
- [ ] Проверен минимальный recovery-runbook.

## 7) JSON-выгрузка пользователя и удаление учётки

- [ ] JSON-архив пользователя формируется корректно.
- [ ] Архив не содержит токены интеграций и другие секреты.
- [ ] Удаление учётки полностью очищает данные пользователя в рамках политики retention.

## 8) OAuth Google/Yandex

- [ ] Регистрация через Google работает.
- [ ] Регистрация через Yandex работает.
- [ ] Логин через Google и Yandex работает.
- [ ] Привязка нескольких методов входа к одной учётке поддерживается.

## 9) Wake-word

- [ ] В активной вкладке есть реакция на wake-word «Ася».
- [ ] После wake-word происходит переход в режим «Слушаю».
- [ ] Команда в режиме «Слушаю» распознаётся и обрабатывается.

## 10) Document-templates (H1–H5)

- [ ] Создание шаблона документа работает.
- [ ] Заполнение шаблона текстом работает.
- [ ] Vision-парсинг VIN/паспорта работает в целевом флоу.
- [ ] Конвертация DOCX → PDF работает.
- [ ] В чат возвращаются два файла: DOCX и PDF.

## 11) Briefings (I2, I3)

- [ ] Утренний брифинг генерируется с релевантным содержимым.
- [ ] Вечерний брифинг генерируется с релевантным содержимым.
- [ ] Архив брифингов за 30 дней доступен пользователю.

## 12) Открытая регистрация и onboarding

- [ ] Этап A (invite-only) зафиксирован и стабилен.
- [ ] Этап B (решение по открытию регистрации) зафиксирован в `docs/decisions.md`.

## 13) Public-facing сайт и юр. часть

- [ ] Публичный лендинг доступен по целевому домену.
- [ ] Юридические документы (минимум ToS и Privacy) опубликованы.

## 14) Стабилизация

- [ ] Security audit завершён без критических замечаний.
- [ ] Нагрузочный тест выполнен, отчёт сохранён в документации.
