# Spaces (Asya 0.3)

Документ описывает пользовательские пространства для Asya 0.3.

## Цель
Разделить контексты работы пользователя и дать per-space контроль памяти и поведения.

## Минимальная модель
- У каждого пользователя есть базовое пространство по умолчанию.
- Пользователь может создавать, переименовывать, архивировать свои пространства.
- Чаты привязываются к пространству (`chat.space_id`).

## Space settings (v0.3)
- `memory_read_enabled`
- `memory_write_enabled`
- `behavior_rules_enabled`
- `personality_overlay_enabled`

## Admin-only пространство
- `Asya-dev` существует как служебное пространство.
- Доступ к `Asya-dev` имеет только администратор.
- Обычный пользователь не видит и не управляет `Asya-dev`.

## Ограничения v0.3
Не входит:
- общие пространства между пользователями;
- сложные роли и ACL внутри пространства;
- внешние источники данных пространства.

## Безопасность
- Все операции со spaces и связанными сущностями обязаны проверять `user_id` владельца.
- Недопустим cross-user доступ к чатам, памяти и правилам через манипуляции с `space_id`.

## Текущая реализация (DB foundation)

Добавлены:
- `spaces` c `user_id`, `is_default`, `is_archived`, `is_admin_only`;
- `space_memory_settings` c флагами:
  - `memory_read_enabled`,
  - `memory_write_enabled`,
  - `behavior_rules_enabled`,
  - `personality_overlay_enabled`.

Также `chats` расширен `space_id` (nullable) для привязки чатов к пространству.

## Backend реализация (текущий этап)

Реализованы backend API и сервисный слой пространств:
- `GET /api/spaces`
- `POST /api/spaces`
- `PATCH /api/spaces/{space_id}`
- `POST /api/spaces/{space_id}/archive`
- `GET /api/spaces/{space_id}/settings`
- `PUT /api/spaces/{space_id}/settings`

Поведение:
- при создании пользователя автоматически создаётся дефолтное пространство `Default`;
- `Base-chat` создаётся в дефолтном пространстве пользователя;
- обычный пользователь не видит `Asya-dev` и не может его создать;
- для admin автоматически создаётся служебное пространство `Asya-dev`;
- `Asya-dev` нельзя переименовать/архивировать обычной операцией.

Безопасность:
- все endpoint-ы работают только в рамках `current_user`;
- доступ к чужим пространствам возвращает `404`.

## Frontend UI (реализовано)

В `ChatPage` добавлен простой UI пространств без отдельного редизайна:
- список пространств;
- создание, переименование, архивирование пространства;
- выбор текущего пространства;
- показ только чатов выбранного пространства;
- блок per-space memory settings:
  - читать память,
  - записывать память,
  - использовать правила,
  - использовать personality overlay.

Дополнительно:
- `Base-chat` явно помечается как базовый (`Base-chat (базовый)`);
- для admin в списке может отображаться `Asya-dev` (если backend вернул);
- для обычного пользователя `Asya-dev` скрывается на уровне backend и дополнительно фильтруется во frontend.
