## FAQ

**Q: Можно ли запускать этапы не по порядку?**
A: 4A должен быть принят до 4B. Внутри 4B — параллельно. 5 — параллельно, после синхронизации 4B. 6 — после 5. 7 — параллельно, после 6. 8 — последний.

**Q: Два агента задели один файл — конфликт?**
A: Первый мерджит. Второй: `git pull origin 1.0-public`, разрешает конфликты, прогоняет тесты. Alembic: `alembic merge heads`.

**Q: Cloud deployment убран из v1.0. Когда VPS?**
A: В версии 2.0. VPS на Windows-сервере, PostgreSQL, S3, CI/CD — всё там. Детализация будет в docs/v2.0-prompts.md при подходе к фазе.

**Q: Что делать с cloud-readiness аудитом?**
A: Он уже сделан (Этапы 1–2 выполнены ранее). /healthz, /readyz, структурированные логи — уже в проекте. Это пойдёт в плюс при VPS-деплое в 2.0.
