# User Guide v1.0

## 1. Начало работы

1. Откройте Asya в браузере по адресу вашего deployment.
2. Зарегистрируйтесь (invite-only или open registration — зависит от текущего режима инстанса).
3. Подтвердите вход и завершите onboarding.

## 2. Базовый сценарий: чат

1. Создайте чат в разделе чатов.
2. Отправьте сообщение.
3. При необходимости загрузите файлы в сессию.
4. Используйте архивирование, чтобы скрыть неактуальные чаты.

## 3. Пространства

- Пространство помогает разделять контексты (работа, личное, проекты).
- Для каждого пространства можно настроить memory behavior.

## 4. Память

- Asya сохраняет факты, правила и эпизоды (в рамках разрешённых настроек).
- Вы можете:
  - редактировать факты;
  - отключать правила;
  - делать snapshot и rollback состояния памяти.

## 5. Дневник

- Поддерживаются текстовые и аудио-записи.
- Для записей строится структурированное представление (topics/decisions/mentions).

## 6. Брифинги

- Доступны утренний и вечерний briefings.
- История брифингов хранится и доступна для просмотра.

## 7. Интеграции

- OAuth-подключения: Google, Yandex (в рамках текущего rollout), а также интеграции, доступные на вашем инстансе.
- Подключения можно отключить в разделе интеграций.

## 8. Голос

- Поддерживается wake-word и режим «Слушаю» (в рамках фазы 1.0).
- Доступность зависит от настроек инстанса и провайдеров речи.

## 9. Выгрузка данных и удаление аккаунта

- В `/me/export` можно запросить JSON-архив данных.
- Ссылка на скачивание одноразовая и ограничена по времени.
- Удаление аккаунта выполняется в два шага: пароль + confirmation token.

## 10. Безопасность

- Интеграционные токены не выдаются в пользовательском export.
- Доступ к данным изолирован по user session.
