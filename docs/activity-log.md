# Activity Log (Asya 0.3)

Документ описывает прозрачную ленту действий ассистента в версии 0.3.

## Цель
Дать пользователю наблюдаемость того, как ассистент использует память, правила и пространство.

## Минимальный набор событий
- создание/обновление/архивация пространства;
- запись факта/эпизода памяти;
- изменение статуса памяти (`confirmed`, `forbidden` и т.д.);
- применение rule/personality в контексте ответа (без раскрытия чувствительных данных);
- создание версии и снэпшота памяти;
- откат изменения памяти.

## Поля события (базово)
- `id`, `user_id`, `space_id` (если применимо), `event_type`, `entity_type`, `entity_id`, `created_at`;
- безопасный `summary` для UI;
- `meta` только без секретов.

## Требования
- Лента строго user-scoped.
- События должны быть пригодны для аудита и отката памяти.
- Лента не должна раскрывать секреты, токены, ключи и приватные документы.

## Границы v0.3
- Не нужна полнофункциональная SIEM/форензика.
- Не нужны внешние экспорты в сторонние системы.

## Текущая реализация (DB foundation)

Добавлена таблица `activity_logs`:
- `user_id`, optional `space_id`, `event_type`, `entity_type`, `entity_id`, `summary`, `meta`, timestamps.

События и сущности типизированы enum-ами; лента индексирована для user-scoped чтения.

## Backend интеграция (реализовано)

`ActivityLog` заполняется при операциях:
- память: создание/обновление/изменение статуса факта, создание эпизода;
- правила: создание/обновление/отключение;
- личность: создание/обновление base и space overlay профиля;
- пространства: создание/переименование/архивация;
- настройки пространства: изменение memory/rules/personality toggles;
- chat runtime: событие `memory_used_in_response` при использовании memory context;
- snapshots: создание snapshot памяти (`memory_snapshot_created`).
- rollback: откат памяти к snapshot (`memory_rollback`).

Безопасность payload:
- `summary` и `meta` хранят только безопасные данные (типы изменений, счётчики, bool-flags);
- в ленту не пишутся полный prompt, токены, API-ключи, секреты и приватные документы.

## API ленты активности

`GET /api/activity-log` поддерживает фильтры:
- `limit` (по умолчанию 100);
- `event_type`;
- `entity_type`;
- `space_id`;
- `date_from` (ISO datetime);
- `date_to` (ISO datetime).

Ответ всегда user-scoped по `current_user`.

## Frontend UI (реализовано)

Добавлена вкладка `Активность`:
- список событий с понятными названиями;
- отображение summary, entity type, пространства и времени;
- фильтры по типу события, типу сущности, пространству и диапазону дат;
- состояния `loading/empty/error`;
- технические ID и чувствительные данные не показываются.
